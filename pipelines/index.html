    Pipelines¶

    Six end-to-end flows make up the system. Each page below walks the code path, the contract that locks its boundaries, and the tests that protect it.

    • Streaming ingestion

      Async TfL polling → tier-2 Pydantic events → Kafka topics → idempotent raw.* writes.

    • dbt warehouse

      Staging models with defensive dedup, marts on stable composite grains, exposures that wire marts to API endpoints.

    • RAG ingestion

      Conditional GET on TfL strategy PDFs → Docling parse → OpenAI embeddings → Pinecone namespace per document.

    • LangGraph agent

      Five typed tools (4 SQL + 1 RAG), a Haiku normaliser for LineId, and a SSE projection for streaming.

    • FastAPI surface

      Six endpoints, all RFC 7807 errors, a single async psycopg pool, and OpenAPI 3.1 as the contract.

    • Next.js frontend

      Three views (Network Now, Disruption Log, Ask), all server components by default, types regenerated from OpenAPI.
