## 4. Open sub-decisions carried into Phase 2

Author already answered the eight big ambiguities. A few sub-decisions remain
that the plan will resolve; flagging them here for transparency:

1. **`make seed` target.** Makefile in spec §10 references
   `scripts/seed_fixtures.py`, but spec §17 limits this WP to a one-off fetcher
   and forbids additional helper scripts, and §6 only lists `fetch_tfl_samples.py`
   as the fixture source. Options:
   - (a) keep the `seed` target pointing at a script created in a later WP, and
     either stub the script now with `raise SystemExit("Implemented in TM-A1")`
     or leave the target documented but non-functional.
   - (b) drop the `seed` target from this WP's Makefile and reintroduce it in
     TM-A1 when the seeding behaviour is actually needed.
   Phase 2 picks option (a) with a stub — it preserves the spec's Makefile
   verbatim and `make help` shows the target without surprises.
2. **Airflow `Dockerfile` content.** Spec says it exists for TM-A5 but does not
   mandate content. Phase 2 proposes a minimal `FROM apache/airflow:2.10.3` base
   with a comment deferring real build to TM-A5.
3. **`airflow/requirements.txt`.** Phase 2 keeps it empty with a comment.
4. **`web/lib/mocks/*.json` content.** Spec says "derived from the OpenAPI
   `example` fields". Phase 2 defines the exact mock JSON to emit.
5. **Running `scripts/fetch_tfl_samples.py`.** Will halt and request confirmation
   before execution, per author answer #5.
6. **`shadcn init` interactive prompts.** Author answer #6 provides the answers
   (Default / Slate / CSS variables=Yes). If the CLI surface has changed and
   those prompt labels drift, Phase 3 reports the divergence and asks before
   picking substitutes.
7. **CI matrix for `dbt-parse`.** Author answer #8 says `dbt-parse = 'dbt parse
   --project-dir dbt --profiles-dir dbt --target ci'` and pass `POSTGRES_*`
   env vars in CI. `dbt/profiles.yml` needs to carry both `ci` and `dev`
   targets; plan will lock in defaults (`ci` → `postgres/postgres/postgres`).
